# JAVLibrary 爬虫 - 403 错误排查指南

## 🔴 问题

爬虫返回 **403 Forbidden** 错误，无法获取页面内容。

```
[2026-05-19 22:38:50] INFO: Fetched (403) <GET https://www.javlibrary.com/cn/vl_mostwanted.php>
```

## 🔍 原因分析

HTTP 403 Forbidden 通常表示：
1. **IP 被封禁** - 网站识别到你的 IP 是爬虫并拒绝访问
2. **代理问题** - 使用的代理 IP 被网站封禁
3. **验证失败** - 无法通过 Cloudflare 或其他防护机制
4. **请求头缺失** - 缺少必要的 HTTP 头信息

## ✅ 解决方案

### 方案 1: 更换代理（推荐）

**问题：** 代理 IP 可能已被网站识别和封禁

**解决：**
1. 在 Clash 或其他代理工具中更换 IP
2. 等待一段时间后重试
3. 尝试使用不同的代理服务商

**配置示例：**
```bash
# .env 文件
PROXY_ENABLED=true
PROXY=http://127.0.0.1:7890  # 尝试不同的端口或 IP
```

### 方案 2: 检查代理连接

首先确认代理本身是否工作：

```bash
# 运行诊断脚本
uv run test_proxy.py
```

**预期输出：**
- 如果得到 403，说明代理 IP 被封了
- 如果连接超时，说明代理不可用

### 方案 3: 使用不同的代理类型

如果 HTTP 代理不行，尝试 SOCKS5：

```bash
# .env 文件
PROXY=socks5://127.0.0.1:1080
```

### 方案 4: 增加延迟，降低请求频率

修改 `javlibrary_scrapling.py` 中的延迟：

```python
# 原来是 2-3 秒，改为 5-10 秒
await asyncio.sleep(10)
```

### 方案 5: 等待 IP 冷却

如果 IP 被临时封禁：
1. 停止爬虫
2. 关闭代理，等待 30 分钟到几个小时
3. 重新连接并测试

### 方案 6: 无代理尝试（仅限中国境外）

如果你在中国境外，可以尝试不使用代理：

```bash
# .env 文件
PROXY_ENABLED=false
```

然后运行：
```bash
uv run javlibrary_scrapling.py
```

## 📊 测试步骤

### 步骤 1: 验证代理连接
```bash
uv run test_proxy.py
```

### 步骤 2: 查看诊断输出
- 状态码为 200 ✅
- 状态码为 403 ❌ - IP 被封
- 超时 ❌ - 代理不可用

### 步骤 3: 根据结果采取行动

| 结果 | 原因 | 解决方案 |
|------|------|--------|
| 403 | IP 被封 | 更换代理/等待冷却 |
| 超时 | 代理不可用 | 检查代理设置/更换代理 |
| 200 但无数据 | 结构改变 | 检查网站更新 |
| 200 有数据 | ✓ 正常 | 运行完整爬虫 |

## 💡 建议

1. **短期解决**
   - 更换代理 IP
   - 增加请求间隔
   - 尝试后夜间运行（流量较低）

2. **长期解决**
   - 使用付费代理服务（质量更好）
   - 配置代理池轮换
   - 实现指数退避重试机制

3. **规范爬虫**
   - 遵守网站的 robots.txt
   - 合理安排爬取时间
   - 不进行过度并发

## 📞 常见问题

### Q: 403 错误会持续多久？
**A:** 通常几分钟到几小时，有时需要更长时间。建议使用新的代理 IP 重试。

### Q: 如何确认是 IP 被封还是其他问题？
**A:** 运行 `test_proxy.py`。如果返回 403，说明代理 IP 被封。

### Q: 能否使用多个代理轮换？
**A:** 可以，但需要修改爬虫代码以支持代理池。

### Q: 有没有更稳定的方法？
**A:** 使用高质量的付费代理服务，或在非高峰时间爬取。
